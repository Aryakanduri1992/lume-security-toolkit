## ‚ú® What's New

### 1. ML Natural Language Normalization

**The Problem We Solved:**
Users had to phrase instructions exactly right for the rule engine to understand them.

**The Solution:**
ML normalization layer that understands varied phrasing and normalizes it to canonical instructions.

**Example:**
```bash
# Before (v0.2.0) - Had to be exact
lume "scan ports on 192.168.1.1"  # ‚úì Works
lume "first give ip 192.168.1.1 then scan"  # ‚úó Fails

# After (v0.3.0) - Flexible phrasing
lume --ml-normalize "first give ip 192.168.1.1 then scan"  # ‚úì Works!
# ML normalizes to: "scan ports on 192.168.1.1"
```

### 2. Key Features

#### ‚úÖ Safe ML Architecture
- ML NEVER generates shell commands
- ML NEVER chooses tools or exploits
- ML NEVER executes anything
- ML only normalizes natural language
- Rule engine remains sole authority

#### ‚úÖ Offline Operation
- Uses spaCy (production-grade NLP library)
- Small English model (12MB)
- No API calls
- No cloud services
- No data transmission

#### ‚úÖ Deterministic Intent Mapping
- 12 predefined canonical intents
- Verb + keyword matching
- Explainable confidence scoring
- No black-box decisions

#### ‚úÖ Intent Validation Layer
- Critical safety check
- Rule engine validates all ML output
- Prevents invalid command generation
- Multiple fallback mechanisms

#### ‚úÖ Comprehensive Logging
- Logs original input
- Logs normalized instruction
- Logs ML confidence score
- Logs fallback reasons
- Full audit trail

### 4. CLI Integration

#### New Flags

**--ml-normalize**
- Enables ML normalization
- Disabled by default
- Opt-in feature

**--ml-confidence**
- Sets confidence threshold
- Default: 0.75 (75%)
- Range: 0.0 - 1.0

#### Usage Examples

```bash
# Enable ML normalization
lume --ml-normalize "first give ip 192.168.1.1 then scan"

# Adjust confidence threshold
lume --ml-normalize --ml-confidence 0.80 "check example.com"

# Default behavior (rule-based only)
lume "scan ports on 192.168.1.1"
```

### 6. Security Guarantees

#### What ML Does
‚úÖ Understands varied phrasing
‚úÖ Extracts intent from sentence structure
‚úÖ Extracts targets (IP/domain/URL)
‚úÖ Normalizes to canonical instructions

#### What ML Does NOT Do
‚ùå Generate shell commands
‚ùå Choose tools or exploits
‚ùå Execute anything
‚ùå Make security decisions
‚ùå Send data to cloud
‚ùå Learn from user data

#### Safety Mechanisms
1. **Confidence Thresholding** - Falls back on low confidence
2. **Rule Engine Validation** - Validates all ML output
3. **Graceful Degradation** - Works without spaCy
4. **Audit Logging** - Full transparency
5. **Opt-in Only** - Disabled by default

## üîÑ Migration Guide

### From v0.2.0 to v0.3.0

**No Breaking Changes!**

All existing commands work exactly as before. ML is an optional enhancement.

**To enable ML:**
```bash
# Install ML dependencies
sudo pip3 install -e ".[ml]" --break-system-packages --upgrade
python -m spacy download en_core_web_sm

# Use ML normalization
lume --ml-normalize "your instruction"
```

**To continue without ML:**
```bash
# Just update Lume
sudo pip3 install -e . --break-system-packages --upgrade

# Use as before
lume "your instruction"
```
